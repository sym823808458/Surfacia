SHAP Visualization Command Reference
===================================

The ``shap-viz`` command provides interactive molecular-level interpretation of machine learning models using SHAP (SHapley Additive exPlanations) values. It creates web-based dashboards for exploring feature importance and molecular structure relationships.

.. important::
   This command requires a trained machine learning model and corresponding molecular structures. Use ``surfacia workflow`` first to generate the necessary files.

Basic Usage
------------

Launch SHAP visualization with training data:

.. code-block:: bash

   surfacia shap-viz -i training_data.csv -x /path/to/structures/

Include test data for validation:

.. code-block:: bash

   surfacia shap-viz -i training.csv --test-csv test.csv -x /path/to/structures/

Command Line Arguments
----------------------

Required Arguments
~~~~~~~~~~~~~~~~~~~

``-i, --input PATH``
    Path to the training set CSV file with detailed feature analysis.
    
    **Format**: CSV file generated by workflow analysis
    **Example**: ``-i /home/data/Training_Set_Detailed_Manual_5feats.csv``
    **Required columns**: SMILES, target values, selected features
    **Note**: This file must contain the features used in your trained model

``-x, --xyz-dir PATH``
    Directory containing molecular structure files (XYZ or PDB format).
    
    **Purpose**: Provides 3D structures for molecular visualization
    **Example**: ``-x /home/project/molecules/``
    **Supported formats**: ``.xyz``, ``.pdb``, ``.fchk``
    **Tip**: Use the workflow output directory for best compatibility

Optional Arguments
~~~~~~~~~~~~~~~~~~

``--test-csv PATH``
    Path to test set CSV file for model validation.
    
    **Purpose**: Compare model performance on unseen data
    **Example**: ``--test-csv /home/data/Test_Set_Detailed_Manual_5feats.csv``
    **Format**: Same structure as training CSV
    **Benefit**: Enables out-of-sample validation and generalization assessment

``--api-key TEXT``
    API key for external visualization services.
    
    **Security**: **Never share your API key in public repositories**
    **Example**: ``--api-key "your-secure-api-key"``
    **Alternative**: Set environment variable ``SURFACIA_API_KEY``
    **Purpose**: Enables advanced molecular rendering features

``--port INT``
    Port number for the web dashboard.
    
    **Default**: ``8051``
    **Range**: 1024-65535
    **Example**: ``--port 8080``
    **Note**: Ensure the port is not blocked by firewall

``--host TEXT``
    Host address for the web server.
    
    **Default**: ``"0.0.0.0"`` (all interfaces)
    **Examples**: 
    - ``--host "localhost"`` (local access only)
    - ``--host "192.168.1.100"`` (specific IP)
    - ``--host "0.0.0.0"`` (network accessible)

Advanced Options
~~~~~~~~~~~~~~~~

``--skip-surface-gen``
    Skip surface generation for faster startup.
    
    **Default**: ``False``
    **Use case**: When surface files already exist
    **Benefit**: Reduces initialization time
    **Trade-off**: May miss updated surface calculations

``--surface-quality TEXT``
    Quality level for surface generation.
    
    **Choices**: ``"low"``, ``"medium"``, ``"high"``, ``"ultra"``
    **Default**: ``"medium"``
    **Impact**: Higher quality increases computation time
    **Example**: ``--surface-quality "high"`` for publication-ready images

``--color-scheme TEXT``
    Color scheme for molecular visualization.
    
    **Default**: ``"default"``
    **Options**: ``"jmol"``, ``"rdkit"``, ``"cpk"``, ``"custom"``
    **Example**: ``--color-scheme "cpk"`` for traditional chemistry colors

``--cache-dir PATH``
    Directory for caching computed surfaces.
    
    **Purpose**: Speeds up repeated visualizations
    **Default**: ``./.surfacia_cache/``
    **Example**: ``--cache-dir /tmp/surfacia_cache/``

Interactive Dashboard Features
-----------------------------

The SHAP visualization dashboard provides multiple interactive panels:

**Main Features**:

1. **Feature Importance Plot**
   - Global feature ranking across all molecules
   - Interactive filtering by feature type
   - Export capabilities for publications

2. **Molecular Structure Viewer**
   - 3D molecular visualization with rotation/zoom
   - Surface property mapping (ESP, electrostatic potential)
   - Interactive atom selection and highlighting

3. **SHAP Waterfall Plot**
   - Individual molecule contribution analysis
   - Base value vs. prediction breakdown
   - Feature-level explanation visualization

4. **Force Plot Visualization**
   - Dynamic SHAP force plots for molecules
   - Interactive feature contribution display
   - Comparative analysis between molecules

5. **Summary Plots**
   - Beeswarm plots for feature distributions
   - Dependence plots for feature interactions
   - Correlation matrices and heatmaps

6. **Model Performance Metrics**
   - R², MAE, RMSE calculations
   - Training vs. test set comparison
   - Cross-validation results display

Navigation Controls
~~~~~~~~~~~~~~~~~~~

**Mouse Controls**:
- **Left Click + Drag**: Rotate molecule
- **Right Click + Drag**: Zoom in/out
- **Middle Click + Drag**: Translate molecule
- **Scroll**: Zoom in/out

**Keyboard Shortcuts**:
- **Space**: Reset view
- **R**: Toggle rotation
- **S**: Toggle surface display
- **L**: Toggle labels
- **F**: Fullscreen mode

Output and Interpretation
------------------------

**Dashboard Access**:

- Mobile: Responsive design for tablets and smartphones

**Generated Files**:
- ``shap_plots/``: Static plot exports
- ``surface_cache/``: Cached molecular surfaces
- ``logs/``: Dashboard activity logs
- ``exports/``: User-generated exports

**Export Options**:
- **PNG**: High-resolution images (300+ DPI)
- **SVG**: Vector graphics for publications
- **PDF**: Report-ready document formats
- **CSV**: Raw data for further analysis

Feature Interpretation
~~~~~~~~~~~~~~~~~~~~~

**SHAP Value Meaning**:
- **Positive**: Feature increases predicted value
- **Negative**: Feature decreases predicted value
- **Magnitude**: Importance/strength of contribution
- **Base Value**: Average prediction across dataset

**Common Molecular Features**:
- **ALIE**: Average Local Ionization Energy
- **ESP**: Electrostatic Potential
- **Surface Area**: Molecular surface descriptors
- **Shape**: Geometric and topological features
- **Atomic Properties**: Element-specific characteristics

**Analysis Strategies**:

1. **Global Importance**
   - Identify most influential features overall
   - Compare feature rankings between datasets
   - Detect systematic biases in model learning

2. **Local Interpretation**
   - Examine individual molecule predictions
   - Understand specific chemical contributions
   - Validate model chemical intuition

3. **Feature Interactions**
   - Explore non-linear relationships
   - Identify synergistic effects
   - Discover mechanistic insights

Troubleshooting
---------------

Common Issues and Solutions:

**Dashboard Won't Start**:
- Check if port is available: ``netstat -an | grep PORT``
- Try different port: ``--port 8080``
- Verify file permissions and paths

**Molecular Visualization Issues**:
- Ensure structure files exist in specified directory
- Check file format compatibility (.xyz, .pdb, .fchk)
- Verify SMILES strings match structure files

**Slow Performance**:
- Reduce dataset size for testing
- Use ``--surface-quality "low"`` for faster loading
- Clear cache: ``rm -rf .surfacia_cache/``

**Memory Issues**:
- Close unnecessary browser tabs
- Use smaller datasets initially
- Increase system RAM if available

**API Key Problems**:
- Verify key validity and permissions
- Check network connectivity
- Use environment variable instead of command line

**File Format Errors**:
- Ensure CSV files have required columns
- Check for encoding issues (UTF-8 recommended)
- Validate data types and formats

Technical Notes
---------------

**System Requirements**:
- **Memory**: 4-8 GB RAM minimum, 16+ GB recommended
- **Storage**: 1-5 GB for large datasets with surfaces
- **Network**: Stable connection for API features
- **Browser**: Chrome, Firefox, Safari, Edge (modern versions)

**Performance Optimization**:
- **Dataset Size**: <1000 molecules optimal for interactive use
- **Surface Generation**: Pre-compute for large datasets
- **Caching**: Enable for repeated analyses
- **Network**: Use wired connection for stability

**Security Considerations**:
- **API Keys**: Treat as passwords, never share publicly
- **Network Access**: Restrict to trusted networks
- **Data Privacy**: Be aware of data sensitivity
- **Firewall**: Configure appropriate port access

**Integration with Other Tools**:
- **Jupyter**: Embed dashboard in notebooks
- **Python API**: Programmatic access available
- **R Integration**: Export data for R analysis
- **Database**: Save results to SQL databases
